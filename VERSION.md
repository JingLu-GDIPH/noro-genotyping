# Norovirus Genotyping Pipeline Version

Version: 2.2.0
Date: 2026-07-08

## 2.2.0

- Replaced the two-pass BLAST design with a single genome BLAST. The top-scoring genome hit fixes the genogroup and, via the precomputed gene coordinate table (`genome_gene_coordinates.tsv`), provides the RdRp/VP1 coordinates used to slice each query. This removes the previous second region-library BLAST and reduces runtime.
- Added FastTree SH-like branch support (`-nome -mllen`) for RdRp/VP1 placement. A genotype is reported as `typed` only when the clade support clears `--bootstrap-threshold` (default 0.75); below the threshold the genotype is still reported but flagged `unsure` and the status becomes `partial_genotype` or `genotype_unsure`.
- Added genomic position annotation for every query (`query_genome_region`, `query_genome_proteins`, `query_genome_coords`) by projecting the best genome hit onto the gene coordinate table.
- Queries that do not overlap RdRp/VP1 are reported with `status=off_region` plus an `off_region_top_references` list of the closest genome references, instead of being forced through the tree step.
- New status values: `genotype_unsure` and `off_region` (in addition to `typed`, `partial_genotype`, `genogroup_only`, `unclassified`).
- New output columns: `rdrp_genotype_confidence`, `rdrp_bootstrap`, `vp1_genotype_confidence`, `vp1_bootstrap`, `query_genome_region`, `query_genome_proteins`, `query_genome_coords`, `off_region_top_references`. Removed: `min-reference-coverage` semantics (replaced by `--min-slice-identity` / `--min-slice-length`).
- New offline helper `download_genome_annotations.py`: downloads GenBank records for the genome references, parses CDS features (ORF1/VP1/VP2), derives RdRp coordinates by BLASTing the region library back against each genome, and writes `ref_seq/genome_gene_coordinates.tsv` plus a `ref_seq/genbank_cache/` of `.gb` files. Required once before running the pipeline.
- New self-contained HTML report (`generate_report.py`, invoked via `genotype_norovirus.py --report` and enabled by default in Nextflow). The report is table-only: a summary of status counts, RdRp and VP1 genotype tables (with SH-like support and typed/unsure confidence), and a genomic-position table that labels every query with its ORF and specific protein (e.g. NTPase, VPg, 3CLpro, 5'UTR) and lists the closest references for off-region sequences.
- Off-region annotation expanded: the gene coordinate table now subdivides ORF1 into its six non-structural cleavage products (p48 / NTPase / p22 / VPg / 3CLpro / RdRp) using conserved relative boundaries, and records 5'/3' UTRs from GenBank. Queries that fall outside RdRp/VP1 are therefore annotated with the actual gene they hit, not just "ORF1".
- Nextflow now stages `genome_gene_coordinates.tsv` into each sample's working `ref_seq/`.

## Algorithm

1. Run a single BLAST of each query against the genotype-labelled near-complete genome references. The top-scoring genome hit fixes the genogroup.
2. Use the best genome hit plus the precomputed gene coordinate table (`genome_gene_coordinates.tsv`) to slice the RdRp and VP1 segments out of the query. Coordinates come from GenBank CDS annotations (ORF1/VP1/VP2) plus RdRp coordinates derived by BLASTing the region library back against each genome.
3. Annotate every query with its genomic position (ORF1/ORF2/ORF3 and overlapping proteins) projected from the best genome hit.
4. For each query that covers a typed region, align the sliced segment to the corresponding regional reference alignment using MAFFT `--addfragments`, then build a FastTree tree with SH-like branch support (`-nome -mllen`).
5. Assign the genotype from the nearest reference tip. Report `typed` when the clade support is at least `--bootstrap-threshold`, otherwise flag the call `unsure`.
6. For queries that fall outside RdRp/VP1, report `off_region` and list the closest genome references instead of forcing a tree placement.

Complete genomes are never used directly for RdRp or VP1 phylogenetic trees; only the coordinate-derived slices are.
